设置

项目名称
默认路径
如果项目中的所有图像都保存在一个文件夹中，请将默认路径设置为该文件夹的位置。默认情况下，VIA应用程序将从该文件夹加载图像。 注意：默认路径 "./" 指示包含 via.html 的应用程序文件还包含此项目中的图像。例如: /datasets/VOC2012/JPEGImages/ 或者 C:\Documents\data\ (注意尾随 / 和 \)
搜索路径列表
如果定义多个路径，则将搜索所有这些文件夹以查找此项目中的图像。我们不推荐这种方法，因为在多个文件夹中搜索图像的计算成本很高。
    区域标签
    默认情况下，图像中的每个区域都使用region-id进行标记。在这里，您可以选择更具描述性的区域标签。
    区域颜色
    默认情况下，每个区域都使用单一颜色绘制。使用此设置，可以为根据区域属性分组的区域指定唯一的颜色。
    区域标签字体
    显示区域标签的字体大小和字体系列。
    预加载缓冲区大小
    图像被预加载在缓冲区中，以允许下一个/上一个图像更平滑的导航。较大的缓冲区大小可能会降低浏览器的整体性能。若要禁用预加载，请将“缓冲区大小”设置为0。
    图像注释编辑器
    当选择一个区域时，图像上的注释编辑器被激活，用户可以更新该区域的注释。默认情况下，此图像注释编辑器放置在选定区域附近。

    未找到文件

    文件名:

    我们推荐你在项目设置中更新默认路径为包含这个图片的文件夹.

    临时的修正解决方案是使用 浏览器的文件选择 来手动定位添加图片。 我们不推荐这个方法，因为每次在VIA应用中加载这个项目，你需要重复这个过程。

    • 开始标记, 选择图片 并且绘制区域
    • 使用 属性编辑器 定义属性 (比如 name) 以及 标注编辑器 使用这些属性来描述每个区域 (比如 cat).
    • 需要帮助, 查看 开始使用 页面和预加载示例: 图像标注 和 面部标注.

    更详细的用户使用指导 (包含截屏和描述) 获取.

    1. 加载图片: 第一步是加载所有你想标注的图像. 在VIA项目中有很多种方式来添加图片. 选择使用于你的方式.
      • 方法 1: 通过浏览器的文件选择，选择本地图片
        1. 点击 项目 → 添加本地文件
        2. 选择想要大的图片点击 打开
      • 方法 2: 通过URL或者绝对路径加载
        1. 点击 项目 → 从URL添加图片
        2. 输入URL 点击确定
      • 方法 3: 从保存在文本文件中的url列表或者绝对路径添加文件
        1. 创建包含URL或者绝对路径(每行一个)的文本文件
        2. 点击 项目 → 从文本文件添加url或者绝对路径
        3. 选择文本文件 点击 打开
    2. 绘制区域:
      • 长方形、圆形和椭圆
        • 按下鼠标左键，拖动鼠标光标，释放鼠标按钮。
        • 若要在现有区域内定义一个点，请在该区域内单击以选择它(如果尚未选中)，现在按鼠标左键，拖放以在现有区域内绘制区域。
        • 要进行选择，请在区域内单击。如果单击点包含多个区域，那么在该位置多次单击会将选择移到这些区域。
      • 点
        • 点击定义点
        • 若要在现有区域内绘制区域，请在区域内单击以选择该区域(如果尚未选定)，现在再次单击以定义该点。
        • 若要选择，请单击(或靠近)现有的点。
      • 点击定义顶点。
      • 按下 [Enter]键结束绘制区域或者按下[Esc]来取消.
      • 如果第一个顶点需要定义在一个现有的区域内，点击区域内选择它(如果还没有选择)，现在再次点击定义顶点。
      • 要进行选择，请在区域内单击。如果单击点包含多个区域，那么在该位置多次单击会将选择移到这些区域。
    3. 创建标注: 有关此步骤的更详细描述, 查看 创建标注: VIA用户指导.点击 视图 → 切换属性编辑器 展示左侧属性编辑区域 并添加所需的文件或区域属性(例如name)。 现在点击 视图 → 切换标注编辑器 在底部显示标注编辑器. 更新每块区域的标注.
    4. 导出标注: 导出json和csv格式的标注, 点击头部菜单栏的 标注 → 导出标注。
    5. 保存项目: 想要保存项目, 点击头部菜单栏的 项目 → 保存。

    Version 2.0.10

    VGG Image Annotator (VIA) is an image annotation tool that can be used to define regions in an image and create textual descriptions of those regions. VIA is an open source project developed at the Visual Geometry Group and released under the BSD-2 clause license.

    Here is a list of some salient features of VIA:

    • based solely on HTML, CSS and Javascript (no external javascript libraries)
    • can be used off-line (full application in a single html file of size < 400KB)
    • requires nothing more than a modern web browser (tested on Firefox, Chrome and Safari)
    • supported region shapes: rectangle, circle, ellipse, polygon, point and polyline
    • import/export of region data in csv and json file format

    For more details, visit http://www.robots.ox.ac.uk/~vgg/software/via/.
